Git Practice Repository

A tiny website built to practice commits, branches, merges, and restores.

What this repo is for

This project is intentionally small so you can make quick edits, commit often, and learn Git by doing. The website is part of the practice: every section is an easy spot to modify and create clear commits.

Branches

  • main — stable “final” site
  • feature-practice — experiment branch

Practice: change one sentence on feature-practice and merge into main.

Commits

  • Edit the “Practice Log” below
  • Update one style rule in styles.css
  • Add a line to notes.txt

Tip: keep commits small — one idea per commit.

Merges

  • Fast-forward merge (simple updates)
  • Normal merge commit (keep branch history)
  • Optional: create a tiny conflict on purpose

Practice: change the same line on both branches and resolve it.

Restores & Undo

  • Restore a file you edited by mistake
  • Revert a commit (safe undo)
  • Reset locally (learn the difference)

Practice: break a CSS rule, then restore the file.

Practice Log

Update these checkboxes as you practice. This section is designed for quick commits.

Quick Commands (for practice)

# create branch + switch
git switch -c feature-practice

# stage + commit
git add .
git commit -m "Update practice log"

# merge into main
git switch main
git merge feature-practice

# restore a file (discard uncommitted changes)
git restore styles.css

# revert a commit (safe undo)
git revert <commit-hash>

You don’t need to run all of these right now—this is just a handy reference on the page.
